# 🚀 TRADE BOT — полный гайд развёртывания в Telegram

От «скачал архив» до «в Telegram работает бот» — примерно 30–40 минут.

## 📦 Что нужно установить на компьютер

Перед началом убедись, что установлены:

- **Node.js 20+** — https://nodejs.org (скачай LTS)
- **Git** — https://git-scm.com
- **VS Code** или любой другой редактор
- Аккаунт на **GitHub** (бесплатный)

Проверь версии в терминале:
```bash
node --version   # должно быть v20.x или выше
npm --version
git --version
```

## 📁 Шаг 1. Структура проекта

Создай на компьютере папку `trade-bot`, внутри неё будет две подпапки:

```
trade-bot/
├── webapp/          ← Mini App (React-фронт)
│   ├── src/
│   │   ├── App.jsx
│   │   ├── main.jsx
│   │   └── index.css
│   ├── index.html
│   ├── package.json
│   ├── vite.config.js
│   ├── tailwind.config.js
│   └── postcss.config.js
│
└── bot/             ← Telegram-бот (Node.js)
    ├── bot.js
    ├── package.json
    └── .env
```

Открой терминал и создай скелет:

```bash
mkdir -p trade-bot/webapp/src trade-bot/bot
cd trade-bot
```

## 🎨 Шаг 2. Настройка фронта (webapp)

### 2.1. Положи файлы

Скопируй файлы, которые я тебе сгенерировал, в соответствующие места:

| Файл у меня в output | Куда положить |
|---|---|
| `trade-app-bot.jsx` | `webapp/src/App.jsx` *(переименуй!)* |

| `webapp-main.jsx` | `webapp/src/main.jsx` |
| `webapp-index.css` | `webapp/src/index.css` |

### 2.2. Поставь зависимости и запусти локально

```bash

npm install
npm run dev
```

Откроется `http://localhost:5173` — должен увидеть Mini App. Цвета, анимации, переключение языка — всё должно работать.

> ⚠ В обычном браузере кнопка «Получить сигнал» работать не будет: она появляется только из-под Telegram. Для теста всего остального достаточно браузера.

### 2.3. Проверка сборки

Перед деплоем проверь что проект собирается:

```bash
npm run build
```

Должна появиться папка `webapp/dist/` — это и есть то, что мы задеплоим.

## 🌍 Шаг 3. Деплой фронта на Vercel (бесплатно, 5 минут)

Telegram **требует HTTPS** для Mini Apps. Vercel его даёт автоматически.

### 3.1. Загрузи проект на GitHub

```bash
cd trade-bot
git init
git add .
git commit -m "initial"
```

Заведи новый пустой репозиторий на github.com (назови `trade-bot`), потом:

```bash
git remote add origin https://github.com/ТВОЙ_USERNAME/trade-bot.git
git branch -M main
git push -u origin main
```

### 3.2. Импортируй в Vercel

1. Зайди на https://vercel.com, войди через GitHub
2. Жми **Add New → Project**, выбери репо `trade-bot`
3. **Root Directory** → нажми **Edit** → выбери `webapp`
4. Framework preset должен сам определиться как **Vite**
5. Жми **Deploy**

Через 1–2 минуты получишь URL вида `https://trade-bot-xyz.vercel.app`. Это твой **WEBAPP_URL** — сохрани.

Проверь: открой этот URL в обычном браузере. Должно показывать Mini App.

## 🤖 Шаг 4. Создание бота в Telegram

### 4.1. Получи токен

1. Открой Telegram, найди **@BotFather**
2. Отправь `/newbot`
3. Введи **имя** бота (отображаемое): например, `Trade Bot`
4. Введи **username** (обязательно заканчивается на `bot`): например, `my_trade_app_bot`
5. BotFather пришлёт **токен** вида `1234567890:AAH...` — скопируй и сохрани

### 4.2. Привяжи Mini App к боту

В @BotFather:

```
/mybots → выбери своего бота → Bot Settings → Menu Button → Configure menu button
```

- **Button text:** `📊 Trade Bot` (то, что будет видно в левом нижнем углу чата)
- **Button URL:** твой `https://trade-bot-xyz.vercel.app` из Vercel

После этого у пользователей в чате с ботом появится кнопка, открывающая Mini App.

### 4.3. (Опционально) Картинка и описание

Всё в @BotFather:
- `/setdescription` — текст, который виден до `/start`
- `/setabouttext` — короткое описание в профиле бота
- `/setuserpic` — аватарка бота
- `/setcommands` — список команд (напиши туда: `start - Запустить бота`)

## ⚙️ Шаг 5. Настройка и деплой бота (Node.js)

### 5.1. Положи файлы бота

Скопируй в `trade-bot/bot/`:

| Файл у меня в output | Куда положить |
|---|---|
| `bot.js` | `bot/bot.js` |
| `package.json` | `bot/package.json` |
| `.env.example` | `bot/.env` *(переименуй)* |

### 5.2. Заполни `.env`

Открой `bot/.env` и впиши свои значения:

```env
BOT_TOKEN=1234567890:AAH...       # от @BotFather
WEBAPP_URL=https://trade-bot-xyz.vercel.app   # от Vercel
SUPPORT_URL=https://t.me/твой_аккаунт
REVIEWS_URL=https://t.me/канал_с_отзывами
```

### 5.3. Локальный запуск для проверки

```bash
cd bot
npm install
npm start
```

В терминале должно появиться `✅ @твой_бот is running`. Открой Telegram, напиши боту `/start` — увидишь приветствие с кнопками. Нажми «🚀 Получить сигнал» — откроется Mini App.

Если всё работает — двигаемся к деплою бота на сервер, чтобы он не отключался, когда закрываешь ноутбук.

### 5.4. Деплой бота на Railway (бесплатно, 3 минуты)

Railway — самый простой способ. 500 часов/месяц бесплатно, чего для Telegram-бота хватит с запасом.

1. Зайди на https://railway.app, войди через GitHub
2. Жми **New Project → Deploy from GitHub repo**, выбери репо `trade-bot`
3. Railway спросит про настройки:
   - **Root Directory:** `bot`
   - **Start Command:** `npm start` (или оставь по умолчанию)
4. Перейди в **Variables** и добавь те же переменные, что в `.env`:
   - `BOT_TOKEN`
   - `WEBAPP_URL`
   - `SUPPORT_URL`
   - `REVIEWS_URL`
5. Жми **Deploy**

Через минуту в логах увидишь `✅ @твой_бот is running`. Всё, бот работает 24/7.

> 💡 Альтернативы Railway: **Render.com** (free tier с засыпанием), **Fly.io**, свой **VPS** (Hetzner от €4/мес).

1. Открой Telegram → найди своего бота по username → `/start`
2. Увидишь приветствие, 4 inline-кнопки + большая «🚀 Получить сигнал»

   - Клик по активу запускает анализ и показывает ВВЕРХ/ШОРТ
   - Кнопка «АНАЛИЗИРОВАТЬ РЫНОК» снизу запускает анимацию с реальными котировками
   - В калькуляторе пересчитываются Мартингейл и симуляция

Если всё работает — **поздравляю, бот в продакшене**.

## 🔧 Траблшутинг

| Проблема | Причина и решение |
|---|---|
| Кнопка «Получить сигнал» не открывает приложение | WEBAPP_URL в .env не совпадает с тем, что в BotFather. Сверь оба места. |
| «Ошибка загрузки» внутри Mini App | Vercel ещё не задеплоил, подожди 1–2 минуты или проверь логи сборки на Vercel |
| Цены всегда fallback, не реальные | Binance/Frankfurter иногда блочат CORS из определённых регионов. Решение ниже (в разделе «Развитие») — вынести fetch на бэк |
| Бот не отвечает на `/start` | Проверь переменные на Railway, особенно BOT_TOKEN. Перезапусти деплой. |
| Tailwind классы не применяются | Забыл `postcss.config.js` или неверный `content` в `tailwind.config.js` |

## 🚀 Что дальше развивать

1. **Вынести fetch цен на бэк** — чтобы не зависеть от лимитов клиента и обойти CORS. Нужен отдельный `server/` с Express + Redis-кэшем + WebSocket на Binance.
2. **Реальный экономический календарь** — подключить парсер ForexFactory XML (бесплатный) на бэке.
3. **Auth через initData** — обязательно до публичного запуска. Проверка HMAC-подписи Telegram на бэке, чтобы знать реального пользователя и не дать подделать user_id.
4. **PostgreSQL** — хранить пользователей, язык, избранные активы, историю сигналов. Сейчас всё в памяти.
5. **Реальные ML-сигналы** — вместо `Math.random() > 0.5` считать на бэке ансамбль `technicalindicators` (RSI + MACD + Bollinger) по историческим свечам.
6. **Платная подписка** — Telegram Stars или Telegram Payments: кнопка «PREMIUM» разблокирует сигналы с уверенностью >85%.
